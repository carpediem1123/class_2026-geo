## 🎮 모듈별 학습 흐름

### MODULE 01-S1 · 직선과 평면의 수직

직선과 평면이 수직일 **필요충분조건**(평면 위 서로 다른 두 직선과 모두 수직)을 탐구합니다.

- **수학실험실**: 7단계 증명을 학생이 직접 완성합니다  
  - 0단계 가정 조건 체크 → 1단계 **증명 목표 세우기** (임의의 직선 k 도입) → 가설 작도 → 수직이등분선 → SSS 합동 → SAS 합동 → 이등변삼각형 결론  
  - 각 단계에 **빈칸 채우기**가 있어 학생이 합동의 대응변·대응각을 직접 찾습니다  
  - "임의의 직선 하나로 모든 직선을 대신한다"는 핵심 발상을, 오답 시 k가 회전하는 애니메이션으로 체험합니다

### MODULE 01-S2 · 삼수선의 정리

화성 안테나를 복구하는 3개의 미션으로 삼수선의 정리 ①②③을 학습합니다.

| 미션 | 스토리 | 삼수선의 정리 |
| :---- | :---- | :---- |
| 미션 1 | 통신 안테나 안정화 | PO⊥α, OH⊥l → **PH⊥l** |
| 미션 2 | 안테나 지지대 복구 | PO⊥α, PH⊥l → **OH⊥l** |
| 미션 3 | 쓰러진 안테나 복구 | PH⊥l, OH⊥l, PO⊥OH → **PO⊥α** |

- 슬라이더로 점 P·H를 조정하며 각도가 90°가 되는 순간을 직접 찾습니다  
- 미션 완수 후 **정리 카드**에서 가정·결론 버튼을 누르면, 3D 화면의 해당 직각 기호가 **색과 크기로 강조**됩니다 (가정=노랑색, 결론=시안색)  
- 발견한 수직 관계를 케이블·지지대 설치 애니메이션으로 시각화합니다

### MODULE 02 · 정사영

세 개의 탭으로 정사영의 뜻과 넓이를 단계적으로 학습합니다.

- **개념탐구**: 패널 각도 θ를 조절하며 그림자 넓이 변화(S′ \= S·cos θ)를 관찰  
- **개념이해**: 정육면체·정사면체 단면의 정사영 넓이를 3D로 탐구 (기본개념 · 심화① · 심화②)  
  - 힌트를 누르면 꼭짓점이 **샛노랗게 빛나며** 그림자(수선의 발)를 따라가도록 안내  
- **미션탐구**: 화성 태양광 패널의 최적 각도(지면 \-15° \+ 태양 30° \= 45°)를 찾는 문제 해결

## 🧑‍🏫 수업 운영 가이드

### 준비물

- **태블릿** 또는 PC(이때, 화면 가로보기 및 학생 1인 1대 권장)  
- 최신 브라우저 (Chrome, Safari 등)  
- Google Classroom (배포·성찰로그 수합용)

### 배포 방법

1. 사이트 링크(또는 QR코드)를 게시합니다.  
2. 학생은 링크 접속 → **사령부(허브)** 화면에서 모듈을 선택합니다  
3. 모듈은 순서대로(**MODULE 01** 삼수선 1단계 및 2단계 → **MODULE 02** 정사영) 진행하는 것을 권장합니다

### 성찰로그 (학습 정리 및 교사 피드백)

각 모듈 완수 후 학생은 **📝 미션 성찰로그**를 작성합니다. 이 기록은 단순한 활동 마무리가 아니라, **교사가 학생의 학습 정도와 사고 과정을 파악하는 자료**로 활용됩니다.

**학생이 남기는 것**

- **사고 유형 선택** : 학생이 개념에 어떻게 도달했는지 (모듈마다 선택지가 다릅니다)  
  - 모듈01(삼수선): 조건을 기호로 정리 / 3D를 돌려보며 확인 / 수직 기호 위치로 직관 포착 / STEP을 따라 증명  
  - 모듈02(정사영): S·cos θ 공식 떠올림 / 슬라이더로 탐색 / 그래프 직관 / 조건 분석  
- **자유 서술** : 자신이 문제를 해결한 과정을 직접 글로 표현

**모듈별 시트 자동 분리**

- 모듈01과 모듈02의 성찰 내용이 다르므로, 응답이 **각각 다른 시트('모듈01' / '모듈02')에 자동 분리** 기록됩니다  
- 학생이 어느 모듈을 완수했는지에 따라 시스템이 알아서 해당 시트로 보냅니다 (학생은 신경 쓸 필요 없음)  
- 수합 항목: `제출시각 · 모듈 · 학번 · 이름 · 사고유형 · 자유서술`

**교사가 얻는 피드백 — 학습 현황 대시보드**

성찰로그가 쌓이면, 구글시트 안의 **📊 대시보드 탭**이 학급의 학습 현황을 자동으로 요약합니다 (상단 메뉴 → 대시보드 새로고침):

- **제출 현황** — 모듈별 제출 인원과 완료 학생 목록으로 *미제출자*를 바로 파악  
- **사고 유형 분포 차트** — 학급이 어떤 방식으로 개념에 접근했는지 막대그래프로 시각화  
  - 네 가지 유형(🔢 논리·공식형 / 🔁 탐색·시행착오형 / 👁 직관·관찰형 / 📋 절차·분석형)이 두 모듈 공통 구조로 정리됩니다  
  - 예: 직관형이 많고 절차형이 적다면 → 증명의 논리적 단계를 보강하는 후속 지도를 계획  
- **자유 서술 모아보기** — 학생들의 서술을 한 화면에 모아, 오개념·막힌 지점을 훑어보고 개별 피드백의 근거로 활용

이렇게 사고 경향(어떻게 접근했나)과 서술(무엇을 이해했나)을 교차해 보면, 단순 정답 채점을 넘어 **학생의 사고 과정 자체를 평가**가 가능합니다. 또한 학번·이름이 함께 기록되어 개별 학생의 학습 과정을 추적할 수 있고, 누가 어디서 어려움을 겪는지 확인해 맞춤 지도를 설계할 수 있습니다.

**설정 방법**: 성찰로그 수합과 대시보드를 사용하려면 `reflectlog_apps_script.gs`를 참고하세요. Apps Script를 스프레드시트에 연결하면 ① 모듈별 시트 자동 분리와 ② 대시보드 자동 생성이 함께 동작합니다. 생성된 웹 앱 URL을 두 모듈 파일의 `SHEET_URL`에 입력하면 됩니다.

### 운영 팁

- **3D 화면은 드래그로 자유롭게 회전**할 수 있습니다. 학생들에게 "여러 방향에서 관찰해 보라"고 안내하면 공간 감각 형성에 도움이 됩니다  
- 태블릿에서 도형이 안 보이면, 화면의 **\[기하(Geo)\] 버튼**으로 렌더링 방식을 전환할 수 있습니다  
- 슬라이더는 **−/+ 버튼**으로 1°씩 미세 조정이 가능합니다 (태블릿 터치 환경 대응)

## ⚙️ 배포 및 갱신 (GitHub Pages)

이 프로젝트는 GitHub Pages로 호스팅됩니다.

- 파일 수정 후 저장소에 push하면 자동 배포됩니다  
- **모든 파일을 함께 배포**해야 정상 작동합니다. 모듈끼리, 그리고 공통 라이브러리에 서로 의존하기 때문입니다:  
  - `three.min.js` — **네 모듈 모두**가 사용하는 3D 렌더링 라이브러리 (필수)  
  - `OrbitControls.js` — `perp_plane_class.html`(직선과 평면의 수직)이 사용  
  - `index.html`(허브)은 세 모듈 HTML로 이동시키므로, 모든 모듈 파일이 같은 위치에 있어야 합니다  
- 일부 파일만 올리면 **3D 화면이 안 뜨거나(three.min.js 누락) 심화 탭이 빈 화면으로 나오는(adv 누락)** 문제가 생깁니다  
- 배포 후 화면이 갱신되지 않으면 **강력 새로고침**(Ctrl+Shift+R / Cmd+Shift+R) 하세요  
- 허브(`index.html`)의 `BUILD` 값이 모든 모듈에 캐시 버전(`?v=`)을 자동으로 붙여, 학생 태블릿이 항상 최신 버전을 받도록 합니다
